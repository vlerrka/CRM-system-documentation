# Нефункциональные требования
|№|Категория|Требование|Приоритет|Обоснование|
|-|-|-|-|-|
|NFR-001|Безопасность|Аутентификация по логину/паролю, хеширование паролей|Must have|Удобство пользователей и базовые требования безопасности|
|NFR-002|Нормативно-правовые|Соответствие Закону Республики Беларусь «О защите персональных данных» (№ 99-З)|Must have|Система должна обеспечивать выполнение требований законодательства Республики Беларусь в области защиты персональных данных |
|NFR-003|Производительность|Время отклика системы: 95% запросов за < 2 секунд |Should have|Комфортная работа всех пользователей|
|NFR-004|Производительность|Одновременная работа 1000 пользователей|Must have|Минимальная планка для старта продаж|
|NFR-005|Надежность|Доступность 99.5% (Uptime) |Should have|Влияет на удержание|
|NFR-006|Надежность|Ежедневное резервное копирование|Must have|Защита от потери данных|
|NFR-007|Емкость|Хранение не менее 10 ТБ данных (материалы курсов, ДЗ учеников, логи) с возможностью расширения|Should have|Видеоматериалы и загружаемые файлы занимают значительный объем|
|NFR-008|Совместимость|Кроссбраузерность (Chrome, Firefox, Safari, Edge, Яндекс) |Must have|Разные браузеры у пользователей|
|NFR-009|Совместимость|API для интеграций (платежи, вебинары, мессенджеры)|Should have|Можно реализовать в следующей версии|
|NFR-010|Масштабируемость|Горизонтальное масштабирование архитектуры |Should have|Важно для будущего роста, но не для MVP|
|NFR-011|Удобство использования|80% новых пользователей (преподавателей и кураторов) должны успешно выполнить базовые сценарии без обращения в поддержку или к документации |Must have|Показатель интуитивности интерфейса; снижение нагрузки на поддержку и сопротивления внедрению |
|NFR-012|Наблюдаемость|Логирование всех значимых событий и ошибок с возможностью поиска и фильтрации|Must have|Позволяет быстро находить причины сбоев, анализировать инциденты без необходимости воспроизведения |
|NFR-013|Наблюдаемость|Отслеживание бизнес-метрик в реальном времени (количество активных учеников, сданных ДЗ, заявок) через дашборд|Could have|Позволяет бизнесу видеть текущее состояние и быстро реагировать на изменения, но не является первоочередным|
|NFR-014|Поддерживаемость|Наличие модульных тестов с покрытием не менее 70%  функционала (журнал, оплаты, запись на курсы)|Must have|Ускоряет разработку новых функций за счет автоматической проверки|
|NFR-015|Поддерживаемость|Документирование API (OpenAPI/Swagger) для всех внешних и внутренних интеграций|Should have|Позволяет быстро подключать новые сервисы и упрощает отладку интеграций|
